## MODIFIED Requirements

### Requirement: 服务端渲染平台页面
系统必须为人力招聘数据分析后台提供服务端渲染的 EJS 页面。

#### Scenario: 访问仪表盘
- **WHEN** 用户打开根路径 `/`
- **THEN** 系统渲染人力招聘数据分析后台的综合首页或招聘负责人看板入口页面

#### Scenario: 访问后台业务模块路由
- **WHEN** 用户打开员工数据管理、招聘目标管理、面试记录管理或综合分析看板下的任一页面路由
- **THEN** 系统渲染对应的业务页面，而不是返回路由不存在错误或 MVP 空状态页面

### Requirement: 静态资源与公共布局
系统必须为后台业务页面提供公共布局和静态资源支持。

#### Scenario: 渲染公共布局
- **WHEN** 任意后台业务页面被渲染
- **THEN** 页面包含可收起的左侧两层导航、顶部面包屑或标题区、右侧内容区域和一致的平台标题

#### Scenario: 提供本地资源
- **WHEN** 浏览器请求 `/public` 下的文件
- **THEN** 系统返回已配置的静态 CSS 或 JavaScript 资源

## ADDED Requirements

### Requirement: 后台两层导航
系统 MUST 提供模块到页面的两层导航结构，并支持左侧导航收起。

#### Scenario: 展示模块页面结构
- **WHEN** 用户访问后台任一页面
- **THEN** 左侧导航 MUST 展示员工数据管理、招聘目标管理、面试记录管理和综合分析看板模块及其子页面

#### Scenario: 收起左侧导航
- **WHEN** 用户点击导航收起按钮
- **THEN** 系统收起左侧导航并扩大右侧内容区域

#### Scenario: 高亮当前页面
- **WHEN** 用户访问某个业务页面
- **THEN** 系统 MUST 高亮当前页面及其所属模块

### Requirement: 统一列表页面布局
系统 MUST 为业务列表页提供统一的筛选、操作、表格和分页布局。

#### Scenario: 渲染列表页结构
- **WHEN** 系统渲染员工、目标或面试记录列表页
- **THEN** 页面 MUST 包含筛选条件区、查询按钮、重置按钮、操作区、数据表格和分页控件

#### Scenario: 默认分页大小
- **WHEN** 用户首次访问任一业务列表页
- **THEN** 系统 MUST 默认每页展示 20 条记录

#### Scenario: 保留筛选状态
- **WHEN** 用户翻页或执行查询
- **THEN** 系统 MUST 保留当前筛选条件并展示对应页数据

### Requirement: 统一导入弹框
系统 MUST 为需要导入 Excel 的页面提供统一导入弹框交互。

#### Scenario: 打开导入弹框
- **WHEN** 用户点击列表页或导入页的导入按钮
- **THEN** 系统展示导入弹框，并按当前数据类型展示允许的导入模式和覆盖范围提示

#### Scenario: 导入成功刷新页面
- **WHEN** Excel 导入成功
- **THEN** 系统关闭导入弹框，展示导入结果，并刷新当前列表或分析数据

#### Scenario: 导入失败展示错误
- **WHEN** Excel 导入失败
- **THEN** 系统在导入弹框中展示失败原因、缺失字段或异常行摘要
