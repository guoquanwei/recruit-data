## ADDED Requirements

### Requirement: 月度招聘目标导入
系统 MUST 支持导入月度招聘目标 Excel，并按目标所属月份覆盖更新该月目标数据。

#### Scenario: 导入月度目标表
- **WHEN** 操作人员上传人才开发目标拆解 Excel 并确认导入
- **THEN** 系统校验 `整体目标` 工作表和必填列，并将该月份现有目标记录替换为本次文件内容

#### Scenario: 目标日拆解入库
- **WHEN** 系统解析目标表中的 `1日` 至 `31日` 字段
- **THEN** 系统 MUST 保留每个日期的日目标，用于计算截止目标和日维度达成

#### Scenario: 目标导入批次记录
- **WHEN** 月度目标导入完成
- **THEN** 系统记录导入文件名、目标月份、覆盖记录数、成功行数、失败行数和导入时间

### Requirement: 目标列表
系统 MUST 提供目标列表页面，用于查询月度目标配置和达成进度。

#### Scenario: 查看目标列表
- **WHEN** 用户打开目标列表页面
- **THEN** 系统按月份、基地、渠道和招聘订单类型展示目标记录，并默认每页 20 条分页

#### Scenario: 展示达成进度列
- **WHEN** 目标列表展示目标记录
- **THEN** 系统 MUST 展示月度目标、截止目标、实际入培、GAP 和达成率

#### Scenario: 筛选目标列表
- **WHEN** 用户选择月份、基地、渠道或输入关键词并点击查询
- **THEN** 系统按筛选条件刷新目标列表

### Requirement: 目标达成计算
系统 MUST 按既定招聘达成口径计算目标达成指标。

#### Scenario: 计算月度目标
- **WHEN** 系统计算目标记录的月度目标
- **THEN** 系统使用该记录 `1日` 至当月最后一天的日目标合计

#### Scenario: 计算截止目标
- **WHEN** 系统计算目标记录的截止目标
- **THEN** 系统使用该记录 `1日` 至统计截止日的日目标合计

#### Scenario: 计算实际入培
- **WHEN** 系统计算实际入培人数
- **THEN** 系统使用一线员工列表对应的组织表员工数据，按入培时间筛选，并按工号去重统计
- **AND** 系统 MUST 先将入培时间标准化为 `YYYY-MM-DD` 后再判断统计月份，兼容数据库日期对象和文本日期
- **AND** 系统 MUST 使用归一后的员工 `base` 对标目标表 `基地`，使用员工 `招聘渠道` 对标目标表 `渠道`

#### Scenario: 补充实际-only 渠道
- **WHEN** 同一目标基地下存在某渠道有实际入培但没有目标行
- **THEN** 系统 MUST 补充该 `基地 + 渠道` 为 `月度目标 = 0` 的达成行
- **AND** 该实际人数 MUST 计入基地合计和整体合计
- **AND** 只有实际基地属于当月目标基地集合时才补充实际-only 渠道，避免无关基地污染目标达成

#### Scenario: 计算达成率
- **WHEN** 系统计算 GAP 和达成率
- **THEN** 系统使用 `GAP = 实际入培 - 月度目标` 和 `达成率 = 实际入培 / 月度目标`，分母为 0 时展示 `0.00%`

### Requirement: 目标达成进度图表
系统 MUST 提供目标达成进度页面，用于展示整体、基地和渠道维度的目标达成情况。

#### Scenario: 查看整体达成
- **WHEN** 用户打开目标达成进度页面并选择月份
- **THEN** 系统展示整体目标、截止目标、实际入培、GAP、达成率和未达成基地数

#### Scenario: 查看基地渠道明细
- **WHEN** 用户查看目标达成进度明细
- **THEN** 系统按渠道顺序 `回流`、`内推`、`渠道社招`、`渠道校招`、`自主社招`、`合计` 展示每个基地的目标和达成指标

#### Scenario: 标识未达成项
- **WHEN** 基地或渠道的达成率低于 100%
- **THEN** 系统 MUST 在页面上以风险样式标识该项
