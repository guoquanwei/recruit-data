## ADDED Requirements

### Requirement: 员工全量导入
系统 MUST 支持从在职员工 Excel 和离职员工 Excel 导入员工数据，并按来源类型全量覆盖更新对应员工记录。

#### Scenario: 导入在职员工表
- **WHEN** 操作人员上传在职员工 Excel 并确认导入
- **THEN** 系统校验必填列，使用导入文件覆盖所有在职来源员工记录，并记录导入批次结果

#### Scenario: 导入离职员工表
- **WHEN** 操作人员上传离职员工 Excel 并确认导入
- **THEN** 系统校验必填列，使用导入文件覆盖所有离职来源员工记录，并将离职表字段归一到统一员工模型

#### Scenario: 导入失败回滚
- **WHEN** 员工 Excel 导入过程中发生字段缺失、解析失败或写入失败
- **THEN** 系统 MUST 回滚本次覆盖更新，并展示失败原因和异常摘要

### Requirement: 员工统一数据模型
系统 MUST 将在职员工和离职员工归一为统一员工数据模型，用于列表、目标达成、人效和漏斗分析。

#### Scenario: 归一在职员工字段
- **WHEN** 系统处理在职员工记录
- **THEN** 系统将 `部门`、`职位`、`员工状态`、`离职日期`、`手机号码`、`招聘渠道`、`渠道名称`、`办公地点` 和 `入培时间` 映射为统一字段

#### Scenario: 归一离职员工字段
- **WHEN** 系统处理离职员工记录
- **THEN** 系统将 `离职前部门` 映射为统一部门字段，将 `离职前职位` 映射为统一职位字段，并将员工状态标记为离职

#### Scenario: 敏感字段展示
- **WHEN** 员工列表展示手机号或证件号相关字段
- **THEN** 系统 MUST 默认展示脱敏值，不在列表中暴露完整个人敏感信息

#### Scenario: 组织表员工基地归一
- **WHEN** 系统从 `common_emp_org_real_day` 读取一线员工或招聘专员
- **THEN** 系统 MUST 使用 `org_path`、`org_name`、`org_level2` 至 `org_level7` 和 `work_location` 共同推断统一 `base`
- **AND** 一线员工 `base` MUST 归一为目标表可匹配的基地名称，例如 `10015升投`、`京东外呼项目`、`辽宁外呼项目`、`江苏基地-淮安`
- **AND** 组织路径包含 `人才开发部` 的招聘专员 MUST 归一为 `人才开发部`

#### Scenario: 组织表一线岗位口径
- **WHEN** 系统查询一线员工列表或计算目标达成实际入培
- **THEN** 系统 MUST 使用 `客服专员`、`客服班长`、`培训期学员` 作为一线岗位口径
- **AND** 系统 MUST 使用 `train_start_date` 作为入培月份口径

### Requirement: 招聘专员列表
系统 MUST 提供招聘专员列表页面，按招聘专员角色口径查询员工数据。

#### Scenario: 查询招聘专员
- **WHEN** 用户打开招聘专员列表页面
- **THEN** 系统展示 `职位 = 招聘专员` 的员工记录，并按默认每页 20 条分页

#### Scenario: 筛选招聘专员
- **WHEN** 用户输入关键词、所属基地、在职状态、招聘渠道或日期条件并点击查询
- **THEN** 系统按筛选条件刷新招聘专员列表

### Requirement: 一线员工列表
系统 MUST 提供一线员工列表页面，按一线岗位口径查询员工数据。

#### Scenario: 查询一线员工
- **WHEN** 用户打开一线员工列表页面
- **THEN** 系统展示符合一线岗位口径的员工记录，并按默认每页 20 条分页

#### Scenario: 一线员工表格字段
- **WHEN** 系统展示一线员工或招聘专员列表
- **THEN** 表格 MUST 包含工号、姓名、在职状态、所属基地、职位、招聘渠道类型、招聘渠道名称、办公地点、入培日期、离职日期和手机号

### Requirement: 员工列表通用交互
系统 MUST 为员工列表提供统一的筛选、查询、重置、导入和分页交互。

#### Scenario: 重置筛选条件
- **WHEN** 用户点击重置按钮
- **THEN** 系统清空当前筛选条件并恢复第一页列表

#### Scenario: 导入后刷新列表
- **WHEN** 员工数据导入成功
- **THEN** 系统关闭导入弹框，展示导入结果，并刷新当前员工列表数据
