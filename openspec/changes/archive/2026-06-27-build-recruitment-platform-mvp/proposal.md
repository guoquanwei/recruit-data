## 为什么

当前仓库已有《人力招聘数据分析后台技术方案文档》，但还没有可运行的应用骨架。需要先搭建一个低门槛、可 Docker 化、适合路演展示的 MVP 平台入口，让后续候选人管理、招聘进度、渠道分析和 Excel 数据接入可以在清晰边界上继续扩展。

## 变更内容

- 新增 Node.js + Express + EJS + SQLite 的人力招聘数据平台 MVP 骨架。
- 新增 `server.js` 作为应用启动入口，负责初始化 Web 服务、模板引擎、静态资源、中间件、数据库和路由挂载。
- 新增基础页面路由和公共布局，使浏览器访问根路径即可进入招聘数据平台。
- 新增 SQLite 文件模式连接能力，支持后续页面和表结构需求扩展。
- 新增 Docker 单容器交付基础，使平台可通过镜像一键启动。

## 能力范围

### 新增能力
- `recruitment-platform-mvp`: 定义人力招聘数据平台 MVP 的启动入口、基础页面、SQLite 文件连接和单容器运行能力。

### 修改能力

## 影响范围

- 影响应用根目录结构，新增 Node.js 项目配置、入口文件、路由、视图、静态资源、数据库初始化和容器化配置。
- 引入 Express、EJS、SQLite 相关运行依赖。
- 本次不设计候选人、招聘进度、渠道分析等业务表结构；后续招聘业务页面和数据分析能力将在该 MVP 骨架上继续扩展。
