# 员工所属基地归类规则

## 1. 规则目的

员工所属基地用于员工列表展示、招聘目标达成分析、基地维度汇总和未匹配数据排查。

由于员工 Excel 中的 `部门` 和 `办公地点` 并不总是直接等同于目标表中的 `基地` 字段，因此系统需要先根据部门路径和办公地点进行统一归类。

## 2. 适用范围

本规则适用于员工数据导入时的 `所属基地` 识别。

当前主要影响：

- 招聘专员列表
- 一线员工列表
- 目标达成进度中实际入培人员的基地归属
- 综合分析看板中的基地维度统计

## 3. 基本识别原则

系统优先使用 `部门` 字段识别所属基地，必要时结合 `办公地点` 辅助判断。

规则优先级如下：

1. 特殊部门优先识别，例如 `人才开发部`
2. 已确认的业务映射优先识别，例如 `韶关基地 -> 南二在线客服项目`
3. 再按基地、项目、办公地关键字识别
4. 如果无法识别到明确基地，则保留办公地点或部门路径，作为未匹配数据后续排查

## 4. 已确认归类规则

### 人才开发部

当部门路径第三级为 `人才开发部` 时，所属基地统一归为：

```text
人才开发部
```

示例：

```text
伽睿集团 / NEO-OPS / 人才开发部 / 北方招聘组 -> 人才开发部
伽睿集团 - NEO-OPS - 人才开发部 - 济南招聘组 -> 人才开发部
```

### 韶关基地

部门或办公地点包含 `韶关基地` / `SG01-韶关基地` 时，所属基地归为：

```text
南二在线客服项目
```

### 北京基地

部门或办公地点包含以下任一信息时，所属基地归为：

```text
忽略
```

识别关键字：

- `北京基地`
- `BJ01-`
- `BJ02-`
- `BJ03-`
- `BJ04-`

说明：北京基地当前先不进入目标达成分析，因此归为 `忽略`。

### 济南实训基地

部门或办公地点包含 `济南实训基地` / `SD02-济南实训基地` 时，所属基地归为：

```text
济南基地-夏都
```

### 合肥基地

部门或办公地点包含 `合肥基地` / `AH01-合肥基地` 时，所属基地归为：

```text
合肥基地
```

### 成都基地

部门或办公地点包含 `成都基地` / `CD01-成都基地` 时，所属基地归为：

```text
成都基地
```

### 宜宾基地

部门或办公地点包含 `宜宾基地` / `YB-宜宾基地` 时，所属基地归为：

```text
宜宾基地
```

### 贵阳

部门或办公地点包含 `贵阳` / `GZ01-贵阳德福中心` 时，所属基地归为：

```text
ITO项目
```

### 新业务运营中心

部门或办公地点包含以下任一信息时，所属基地归为：

```text
新业务运营中心
```

识别关键字：

- `成都中行`
- `重庆中行`
- `重庆外呼项目`
- `ZH-成都中行`
- `CQ-重庆中行`
- `CQ-重庆外呼项目`

## 5. 其他已实现识别规则

以下规则为系统当前已实现的常规识别逻辑：

| 识别关键字 | 归类结果 |
| --- | --- |
| `联通河北`、`河北基地`、`石家庄` | `联通河北` |
| `联通天津`、`天津` | `联通天津` |
| `江苏` + `淮安` | `江苏基地-淮安` |
| `江苏` 或 `南京` | `江苏基地-南京` |
| `湖南` + `空港` | `湖南基地-空港` |
| `湖南` + `荷花` | `湖南基地-荷花` |
| `济阳` | `济南基地-济阳` |
| `夏都` | `济南基地-夏都` |
| `长春` + `外呼` | `吉林外呼项目` |
| `长春` | `长春热线项目` |
| `10015`、`15升投` | `10015升投` |

## 6. 未匹配处理

如果以上规则都无法识别：

1. 优先使用 `办公地点`
2. 如果办公地点为空，则使用完整部门路径

这类结果通常需要在分析页或数据校验中作为“未匹配基地”排查。
