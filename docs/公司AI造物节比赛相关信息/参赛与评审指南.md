## §2 活动开始前的准备（强烈建议）

这是**业务 + 技术 + AI 三者协作**的活动。配合默契与否，直接决定你们 48 小时能跑多远。**别把热身工作留到 7-04 9 点之后**——48 小时是用来"做出有价值的东西"的，不是用来配环境的。

### 业务同事和技术同事，**请提前一起做完**这张清单

- 开发机已装好基础环境（IDE、Git、Node 或 Python 等）
- AI 编码工具至少装好一个并跑通——**推荐 Claude Code / Codex / ZCode**（这三种的对话记录平台能自动解析）

  - 确认能登录
  - 跑一段最小对话验证
- 大模型 API key 已申请到位

  - 用 GLM：智谱 Coding Plan 已经买好，可以零成本用
  - 用 Claude / GPT / DeepSeek：提前申请 key 并测试一段最小对话；token 费用公司报销
- **Docker 已装好**（Docker Desktop / Docker Engine），能跑 `docker build` 和 `docker push`
- 一起跑通"hello world 镜像 → docker push 到内网仓库"的最小链路（让技术同事带业务同事走一遍流程）
- 技术同事读过 [《镜像构建与部署指南》](https://hack-platform.jiarui.net.cn/docs/deploy) 和 [《技术要点说明》](https://hack-platform.jiarui.net.cn/docs/tech)，**尤其看懂"数据库怎么连、其他环境变量放哪儿"**
- 公司邮箱能正常收件（用于平台登录链接，7-03 晚会收到）

> 💡 不会装 Docker / 配 API key / 写 Dockerfile？**这些事问 AI 都能 5-10 分钟搞定**，正是热身的好机会。预演一遍，比赛当天就有底气。

### 业务、技术、AI 三角配合的"角色分工"建议

| 角色               | 你最该做的事                                                                                             |
| ------------------ | -------------------------------------------------------------------------------------------------------- |
| **业务同事** | 主导题目选型；对 AI 反复**澄清需求和业务规则**；**写测试验证说明**；负责"业务价值"的讲解视频 |
| **技术同事** | 主导技术选型；跟 AI 协作完成代码；保证部署可用；解决环境/报错问题                                        |
| **AI**       | 写代码、改代码、生成测试、写文档、解释错误——但**不替代你们的判断**，需求边界要你们自己定         |

最容易翻车的搭配是：**业务同事"等技术同事写完"** + **技术同事"瞎猜业务需求"**。两边都要**主动跟 AI 对话**，互相对齐。建议每 2-4 小时碰一次进度，对齐方向。

### 核心组（部门总监）独立参赛者

你没有人类搭档，**AI 就是你的搭档**。同样的提醒：提前装好工具、申请 key、跑通最小 docker 链路。不熟的部分都可以问 AI，但**别把工具初始化留到当天**。

### 题目选型小提示

- 选你**真正懂业务**的领域（财务、HR、销售、研发流程……任何一个），因为业务价值要靠你"讲清楚"才有分
- 范围不要贪大，48 小时能跑通一个**主流程闭环**比铺一堆半成品强
- 优先选**有数据支撑**的痛点（"过去 3 个月平均每周耗时 X 小时"），收益锚点清晰评委才信

## §3 你需要交什么

全部通过平台提交，每一项都有专门的页面引导。

### 必交 ✅

#### 1\. 一个能跑起来的应用

平台会给你的小组分一个专属地址（形如 `hack-team-007.jiarui.net.cn`），评委和 AI 都通过这个地址访问你的成果。平台会给你详细的"如何上传"指引；不会的让你的 AI 工具帮你（这正是 AI 原生的体现）。

#### 2\. 测试验证说明 ⭐【最容易被忽略，但**强烈影响分数**】

一份说明"怎么用你这个应用"的文档：

- 测试账号、密码（如果有登录）
- 关键操作路径（"先点 A → 输入 B → 看到 C 才说明功能跑通"）
- 你最想被评估的核心场景是什么

**为什么强调它**：AI 评估会用自动浏览器按这份说明实测你的应用。**没有这份说明，AI 不知道你的核心价值在哪儿，就会少给分**。简单写，但要写清楚。

#### 3\. 讲解视频（5-10 分钟）

⏰ **截止时间是 7-07 10:00**，比其他材料晚 24 小时——你可以**直接拿当天路演的腾讯会议录像作为视频**，不必再单独录一份。

不要求剪辑，**朴素胜过花哨**。视频里讲清楚四件事就够了：

- 你解决的是什么业务问题，为谁解决？
- 收益是什么？能省多少时间 / 多少钱 / 多少错？
- 应用怎么用（屏幕录制或现场演示走一遍主流程）
- AI 怎么帮你完成的（哪些环节让 AI 主导了）

格式：mp4 或 mov，≤ 500MB，≤ 15 分钟。

> 💡 推荐做法：**路演时就把腾讯会议开启云录制 / 本地录制**，结束后下载下来直接上传，省一道工序。如果路演表现不理想，也可以路演后单独再录一份，反正 7-07 10:00 前你随时可以替换。

#### 4\. AI 对话记录

你和 AI 的全程对话日志（`.jsonl` 文件）。这份证明你"AI 原生程度"分数的核心证据，**不要挑选、不要剪辑、不要删**。

主流工具的对话记录位置：

- **Claude Code**：`~/.claude/projects/<项目目录>/*.jsonl`
- **Codex CLI**：`~/.codex/sessions/年/月/日/rollout-*.jsonl`（Windows: `%USERPROFILE%\.codex\sessions\...`）
- **ZCode**：`~/.zcode/cli/rollout/model-io-sess_*.jsonl`

⚠️ 不要给 Codex 加 `--no-persist` 参数（会跳过日志保存）。

可以传**多个文件**，48 小时里跑了几个 session 就传几个。平台会自动转成 markdown 格式给 AI 看。

#### 5\. 题目说明文档

一份简洁的 markdown 文档，把"你做了什么、解决什么问题、有什么亮点"写清楚。专家和 AI 都会看。

### 可选

#### 6\. 代码仓库地址

如果你愿意公开仓库，平台会拉取统计代码规模、提交频次，作为"完成度"和"AI 原生程度"的辅助证据。可选，不影响主流程。

## §4 评分规则

### 4 个维度，每维度 0-10 分

| 维度                  | 权重 | 评什么                                                          |
| --------------------- | ---- | --------------------------------------------------------------- |
| **业务价值**    | 40%  | 是不是真问题 / 收益是否清晰 / 能不能立刻或短迭代后投入工作使用  |
| **AI 原生程度** | 30%  | AI 在需求、设计、写代码、测试各环节的参与深度；人机协作是否高效 |
| **完成度**      | 20%  | 应用能不能跑、有没有 bug、关键路径是不是完整                    |
| **创新性**      | 10%  | 思路、玩法、AI 使用方式是否有新意                               |

### 谁打分 + 怎么汇总

- **3 个 AI 模型** 各自独立打分（Claude Sonnet 4.6、DeepSeek V4 Pro、GPT-5）
- **4 位专家** 各自独立打分
- 最终分 = `0.4 × AI 三模型均分 + 0.6 × 4 专家均分`

### 两个赛道分别排名

- **核心成员组**（部门总监独立参赛）
- **员工组**（业务 + 技术配对参赛）

两组互不竞争，各自评奖。

## §5 给你的建议：怎么拿高分

不少同事会问"我该往哪儿用力"。给四条具体建议：

### ① 业务价值：用**视频**把价值讲透

视频里讲清楚：**为谁、解决什么问题、收益怎么估算**。光做出 demo 不够，要有可量化的价值锚点（"目前 X 流程每周 2 小时，预计能减到 20 分钟"），评委一听就知道你想得有多深。

### ② AI 原生程度：留完整对话记录

让 AI 参与到**需求澄清、方案设计、写代码、自测和重构**的多个环节，而不是只用 AI 写代码片段。对话记录全量上传，连"和 AI 反复对齐需求"、"让 AI 帮你写测试用例"、"让 AI 自检 bug"这些过程都是加分项。

**不要修饰对话**——评分依据的是"过程的真实样子"，不是"最终的精华片段"。

### ③ 完成度：测试验证说明要可执行

测试说明里写的每一步，**AI 都会拿 Playwright 真的去跑**。所以：

- 路径要具体（"点首页右上角'新建任务'按钮"，不是"打开新建任务")
- 账号能登（别忘了把测试账号填上）
- 关键功能能跑出可观察的结果（有数据变化、有提示、有跳转）

### ④ 创新性：小细节也算

不需要"颠覆性创新"。一个意想不到的 AI 使用方式、一个让人眼前一亮的交互细节、一个特别的解决思路——都算。不用刻意憋大招。

## §6 AI 是怎么评的（给好奇的同事）

你不需要理解这部分，但有同事问起就放在这儿：

1. AI 会**真的打开你的应用**，按你的测试验证说明一步步操作、截图、记录报错
2. AI 会**听你的视频**：转写音频 → 修正错别字 → 形成带时间戳的讲解稿
3. AI 会**看你的对话记录**：分析你怎么和 AI 协作
4. AI 会**看你的题目说明和代码仓库（如果提供）**
5. 3 个 AI 模型各自独立给 4 个维度打分，每个分数都要给**理由**和**证据引用**（哪张截图、哪段视频、哪段对话）
6. 专家能看到 AI 的分和理由，但**独立打分**，互不影响

## §7 常见问题

**Q：我可以用什么 AI 工具？** A：任意。但推荐 Claude Code / Codex / ZCode，因为这三种的对话记录平台能自动解析。其他工具的对话需要你自己整理成 markdown 上传。

**Q：用哪个大模型？** A：任意。token 费用公司报销（保留消费记录）。智谱 Coding Plan 已经买好，可以直接用 GLM 系列零成本。

**Q：我必须会 Docker 吗？** A：需要把应用打包成 Docker 镜像才能部署。**但不会写 Dockerfile 没关系，让 Claude Code / Codex / ZCode 帮你写，10 分钟搞定**——这正是 AI 原生的体现。

**Q：锁定时间能不能延长？** A：不能。所有组同时锁定保证公平。

**Q：部署失败怎么办？** A：平台会显示失败原因（Pod 日志摘要），把它复制给你的 AI 工具让它帮你 debug。

**Q：可以反复部署吗？** A：可以，48 小时内任意次。每次部署会替换上一个版本，AI 评估的是锁定时刻的版本。

**Q：视频什么时候交？必须在 7-06 09:00 前吗？** A：不用。视频可以晚到 **7-07 09:00**（比其他材料晚 24 小时）。最简单的做法是路演时把腾讯会议开启录制，结束后下载录像直接上传。如果想路演后单独重录也行，反正 7-07 09:00 前你随时可以替换。

**Q：路演必须用提前录好的视频还是现场讲？** A：腾讯会议路演时**现场讲**，每组约 10 分钟。讲完整理录像，第二天（7-07）9 点前传到平台即可。

**Q：视频一定要剪辑得很精致吗？** A：不需要。重点是把业务价值和 AI 协作过程讲清楚，**朴素胜过花哨**。屏幕录制 + 实时口述就够。

**Q：对话记录里会有敏感信息怎么办？** A：**评委能看到原文**。所以：用脱敏数据写测试用例，不要把真实客户信息、公司内部口令、token 等敏感内容贴到对话里。

**Q：我是业务人员不会写代码怎么办？** A：你的技术搭档负责打通技术链路，你负责**需求澄清 + 跟 AI 对话 + 业务价值阐述**。配合好的组分数不会低。**这正是 AI 原生协作的考点**。

**Q：核心组（部门总监）独立参赛，技术不熟怎么办？** A：同上——让 AI 帮你写。你的核心价值是**对业务的判断力 + 跟 AI 协作的能力**，不是手写代码的速度。

**Q：题目能跨业务线吗？** A：可以。题目范围依组委会公布的指引为准。

**Q：路演时讲不完怎么办？** A：每组约 10 分钟。视频已经提前提交，路演主要是 Q&A 和补充说明。讲不完也没关系，评委会回看视频。
