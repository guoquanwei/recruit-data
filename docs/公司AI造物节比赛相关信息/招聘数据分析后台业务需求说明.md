# 招聘数据分析后台业务需求说明

## 一、项目背景

当前招聘业务数据主要分散在多份 Excel 文件中，包括在职员工表、离职员工表、人才开发目标拆解表、面试记录表等。招聘负责人在查看月度目标达成、基地缺口、渠道占比、自主社招人效和面试转化情况时，需要人工汇总多个数据源并反复计算。

这种方式存在统计效率低、数据口径不统一、报表更新不及时、问题定位困难等问题。随着招聘管理对过程监控和经营分析的要求提高，需要建设一个统一的招聘数据管理与分析后台，将日常 Excel 数据沉淀为可查询、可分析、可追踪的系统化数据。

## 二、当前业务问题

当前业务侧主要存在以下问题：

1. 招聘数据分散在多份 Excel 中，缺少统一的数据查看入口。
2. 月度目标、截止目标、实际入培、GAP、达成率等指标依赖人工计算，效率低且容易出错。
3. 基地和渠道的招聘缺口无法及时暴露，管理者难以及时判断风险。
4. 招聘负责人看板依赖人工整理，更新周期长，无法快速支撑经营决策。
5. 自主社招人效数据统计单一，仅展示达成标准，缺少风险提示。
6. 面试记录与最终入培结果没有形成有效关联，无法清晰判断面试后段转化情况。
7. 数据导入、覆盖、更新过程缺少记录，后续核对和追溯成本较高ahj

## 三、网站定位

本网站是面向招聘负责人、招聘运营人员和相关管理人员的招聘数据管理与分析后台。

系统通过统一导入员工数据、招聘目标数据和面试记录数据，自动生成目标达成、基地风险、渠道占比、自主社招人效和招聘漏斗等分析结果，帮助业务侧提升招聘过程管理效率和数据决策能力。

## 四、使用对象

招聘负责人：查看整体招聘目标达成情况、基地风险、渠道结构和自主社招人效，辅助经营决策。

招聘运营人员：核对导入结果、排查异常数据，保障招聘统计口径一致。

招聘/渠道经理：明确当前达成情况和团队存在的风险，及时调整招聘动作。

## 五、数据来源与用途

系统首期数据主要来自业务上传的 Excel 文件。

| 上传文件           | 包含信息                                                                     | 系统用途                                                                                    |
| ------------------ | ---------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| 在职员工表         | 当前在职员工的工号、姓名、部门、职位、基地、招聘渠道、入培时间、手机号等信息 | 用于统计实际入培人数、在职员工明细、招聘专员规模、目标达成情况和自主社招人效                |
| 离职员工表         | 已离职员工的原部门、原职位、离职时间、招聘渠道、入培时间等信息               | 用于补全历史入培数据，避免只统计在职员工导致实际达成被低估，同时支持流失和 7 天游效相关分析 |
| 人才开发目标拆解表 | 每月各基地、各渠道、各订单类型的招聘目标和每日目标拆解                       | 用于计算月度目标、截止目标、实际入培、GAP、达成率、基地风险和渠道缺口                       |
| 面试记录表         | 候选人、电话、职位、反馈日期、反馈结果、面试官、渠道标签等信息               | 用于查看面试反馈明细，并分析面试反馈到入培的后段转化情况                                    |

三类数据的关系可以理解为：

```text
目标数据：应该完成多少
员工数据：实际完成多少
面试数据：过程转化如何
        ↓
目标达成、基地风险、渠道分析、自主社招人效、招聘漏斗
```

## 六、核心功能范围

员工数据管理：支持在职员工表和离职员工表导入，统一管理员工基础信息，支持招聘专员列表和一线员工列表查询。

招聘目标管理：支持月度招聘目标导入，展示月度目标、截止目标、实际入培、GAP 和达成率。

面试记录管理：支持面试记录导入和查询，沉淀候选人面试反馈明细。

目标达成进度分析：按整体、基地、渠道维度展示招聘目标完成情况，识别未达成项。

综合分析看板：面向招聘负责人展示整体目标、实际入培、GAP、达成率、未达成基地数、渠道占比和自主社招人效等核心指标。

基地风险分析：识别未达成基地，并展示主要缺口渠道，帮助业务侧定位问题来源。

自主社招人效分析：按招聘专员阶段统计参培达成、7 天达成、招聘规模和人效，辅助识别达标人员和风险人员。

招聘漏斗分析：基于面试反馈和员工入培数据，分析面试反馈到入培的后段转化情况。

## 七、使用与维护机制

系统由招聘运营人员负责维护基础数据。招聘运营每周上传并更新一次招聘相关 Excel 文件，包括在职员工表、离职员工表、人才开发目标拆解表和面试记录表，确保系统中的招聘数据保持最新。

招聘负责人和相关管理人员每周基于系统看板进行招聘复盘，重点查看目标达成情况、未达成基地、渠道缺口、自主社招人效和面试转化情况，并据此调整后续招聘动作。

整体使用闭环如下：

```text
招聘运营每周上传数据
        ↓
系统自动计算招聘指标
        ↓
招聘负责人每周查看看板
        ↓
复盘目标达成、基地风险、渠道缺口、人效表现
        ↓
调整招聘策略和人员管理动作
```

## 八、业务价值

建设该网站后，业务侧可以获得以下价值：

1. 减少人工汇总 Excel 和手工制作报表的工作量。
2. 统一招聘目标、实际入培、渠道、人效等核心指标口径。
3. 提高招聘负责人查看经营数据的效率。
4. 及时发现未达成基地、缺口渠道和异常数据。
5. 支持从月底复盘转向过程预警和过程管理。
6. 帮助管理者识别自主社招人员产出差异，提升人效管理能力。
7. 让招聘数据导入、分析和追溯过程更加清晰。

## 九、首期建设边界

首期系统以招聘数据管理和分析看板为主，不建设复杂业务流程。

首期不包含以下内容：

1. 不接入外部 HR 系统、ATS 系统或在线数据库同步。
2. 不建设登录、权限、审批流和多租户能力。
3. 不支持业务数据在线手工编辑，数据来源以 Excel 导入为主。
4. 招聘漏斗首期只覆盖“面试反馈到入培”的后段分析，不包含投递、邀约、到面等前链路数据。
5. 不引入复杂图表平台，优先满足核心数据查看和经营分析诉求。

## 十、预期效果

通过本系统，招聘负责人可以在统一后台中查看招聘目标完成情况、基地风险、渠道贡献、自主社招人效和面试转化情况。业务运营人员可以通过每周上传标准 Excel 文件完成数据更新，系统自动生成分析结果，减少重复手工统计，提高招聘管理效率和数据准确性。

系统上线后，将形成“数据上传 - 自动分析 - 周度复盘 - 行动调整”的管理闭环，帮助招聘业务从人工报表管理逐步转向数据化、过程化的招聘经营管理。
