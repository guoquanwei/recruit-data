# 测试验证说明

## 1. 目标

本文用于验证招聘数据分析后台的核心链路是否可用，重点覆盖：

- 一线员工数据对标目标表后的目标达成计算。
- 基地风险分析默认进入风险最大基地。
- 自主社招人效、招聘专员规模和明细数据展示。
- 页面切换后的性能体验。

## 2. 环境准备

### 2.1 启动服务

在项目根目录执行：

```bash
npm install
npm run dev
```

默认访问地址：

```text
http://localhost:3000
```

### 2.2 数据前置条件

需要确认数据库中已存在以下数据：

- `recruitment_targets`：已导入目标表，例如 `2026-05`、`2026-06`。
- `common_emp_org_real_day`：已有一线员工和招聘专员组织表数据。
- `interview_records`：如需验证漏斗诊断和自招人效明细，需要有面试记录数据。

关键字段口径：

- 一线员工岗位：`客服专员`、`客服班长`、`培训期学员`。
- 招聘专员岗位：`招聘专员`、`初级招聘主管`。
- 一线员工入培时间：`train_start_date`。
- 招聘渠道：`hr_channel_type`。
- 自主社招招聘专员归属：`hr_channel_name` 中 `+` 前的姓名。

## 3. 页面操作步骤

### 3.1 人才开发运营看板

1. 打开左侧菜单 `人才开发运营看板`。
2. 选择月份，例如 `2026/05`。
3. 查看顶部核心指标：
   - 目标达成率。
   - 自主社招占比。
   - 自招 7天人效。
   - 自招团队规模。
4. 查看 `整体目标达成` 列表，确认各基地目标、实际、GAP 和达成率展示正常。
5. 查看 `自招人效` 区域，确认整体、试用期、正式期展示的是：
   - 主值：满 7 天人效。
   - 辅助值：入培人效。

期望结果：

- `2026/05` 示例中，整体目标应为 `424`。
- 实际入培应接近线下口径，当前库表验证值为 `381`。
- 自招 7天人效应和自招人效页整体 7天人效一致。

### 3.2 基地风险分析

1. 打开左侧菜单 `基地风险分析`。
2. 选择月份，例如 `2026/05`。
3. 不手动选择基地，直接查看页面默认状态。

期望结果：

- 顶部 `基地` 筛选自动显示风险最大的基地，例如 `江苏基地-淮安`。
- 顶部 `渠道` 筛选保持 `全部`。
- 页面主体默认展示该基地的多渠道岗位风险。
- 右侧 `漏斗诊断与提升方案` 默认进入该基地内风险最大的渠道和批次。

重点检查：

- `江苏基地-淮安` 在 `2026/05` 下示例指标：
  - 月目标：`150`。
  - 已入职：`103`。
  - GAP：`-47`。
  - 达成率：`68.67%`。
- 渠道筛选不应自动跳成 `自主社招`，但右侧诊断内部可以默认分析 `自主社招` 风险。

### 3.3 自主社招人效

1. 打开左侧菜单 `自主社招人效`。
2. 选择月份，例如 `2026/05`。
3. 查看核心指标卡片：
   - 整体人效。
   - 试用期人效。
   - 正式期人效。
   - 7 天人效达标人数。
   - 招聘团队规模。
4. 查看下方 `自主社招招聘专员达成` 表格。
5. 点击参培达成或 7 天人效数字，确认可以打开明细弹窗。

期望结果：

- 页面不应为空。
- `2026/05` 当前库表验证值：
  - 自招团队规模：`28`。
  - 自招参培人数：`220`。
  - 满 7 天人数：`125`。
  - 整体入培人效：`7.9`。
  - 整体 7天人效：`4.5`。
- 招聘专员明细应有数据，例如 `曹洋`、`陈植`、`董孜婷` 等。

### 3.4 目标达成进度

1. 打开左侧菜单 `目标达成进度`。
2. 选择月份，例如 `2026/05`。
3. 查看整体和各基地达成。
4. 检查有实际但无目标的渠道是否进入基地合计。

期望结果：

- 实际入培基于一线员工列表数据计算。
- 同一目标基地下，`回流` 等没有目标行但有实际入培的渠道应补充为 `月度目标 = 0` 的渠道行。
- 这些实际人数应计入基地合计和整体合计。

## 4. 核心业务规则

### 4.1 一线员工对标目标

- 数据源：优先使用 `common_emp_org_real_day`。
- 组织字段：使用 `org_path`、`org_name`、`org_level2` 至 `org_level7`、`work_location` 共同推断基地。
- 基地归一后再对标目标表 `基地`。
- 目标对标维度：`月份 + 基地 + 招聘渠道`。
- 人数按 `工号` 去重。
- 日期需先标准化为 `YYYY-MM-DD`，再判断月份。

常见归一示例：

| 组织表特征 | 归一基地 |
| --- | --- |
| `河北基地 > 10015升投 > 前台` | `10015升投` |
| `长春基地 > 京东外呼项目` | `京东外呼项目` |
| `辽宁外呼项目`、`辽宁外呼` | `辽宁外呼项目` |
| `江苏` + `淮安` | `江苏基地-淮安` |
| `韶关基地` | `南二在线客服项目` |
| 组织路径包含 `人才开发部` | `人才开发部` |

### 4.2 招聘专员规模和人效

- 招聘专员规模：
  - 组织路径包含 `人才开发部`。
  - 职位为 `招聘专员` 或 `初级招聘主管`。
  - 与统计月存在在岗交集。
  - 排除统计月之前已离职人员。
- 阶段判断：
  - 统计月最后一天入职满 6 个月为 `正式期`。
  - 未满 6 个月为 `试用期`。
  - 优先使用 `入职日期`，缺失时使用 `入培时间`。
- 候选人归属：
  - 只统计 `招聘渠道 = 自主社招` 的一线员工。
  - `渠道名称` 中 `+` 前姓名匹配招聘专员姓名。
- 人效公式：
  - `参培人效 = 参培达成 / 招聘规模`。
  - `7天人效 = 满7天达成 / 招聘规模`。
- 满 7 天达成：
  - `入培时间 + 7 天 <= 统计截止日`。
  - 7 天节点前未离职。
- 页面展示：
  - 综合看板和自招人效页主口径使用 `7天人效`。
  - `参培人效` 作为辅助指标展示。

## 6. 性能验证

页面切换优化后，缓存命中时应明显降低等待时间。

已验证参考值：

- 自主社招人效页首次加载：仍受数据库全量查询影响，可能较慢。
- 自主社招人效页二次切换：约 `1s` 内。
- 人才开发运营看板缓存后切换：约 `1s` 内。

如果首次加载仍明显卡顿，后续优化方向：

- 执行 `scripts/optimize-indexes.sql` 中的索引。
- 将一线员工全量查询改为按统计月份裁剪。
- 将面试记录按月份缓存或按页面所需字段裁剪。
