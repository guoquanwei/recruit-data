# 人才开发目标拆解表内容说明

## 1. 表格基本信息

- 表格名称：人才开发目标拆解表
- 文件命名格式：`人才开发目标拆解-X月-月日.xlsx`
- 文件命名示例：`人才开发目标拆解-4月-0514.xlsx`
- 工作表名称：`整体目标`
- 数据范围：按月份、基地、招聘渠道、招聘订单类型配置人才开发招聘目标及每日目标拆解
- 数据规模：46 条目标记录，39 个字段
- 表头位置：第 1 行
- 数据形态：目标配置宽表，包含月度维度、组织维度、渠道维度、留存率目标、招聘目标及 1 日至 31 日每日目标

## 2. 表格用途

该表用于记录指定月份各基地、各招聘渠道的人才开发目标，包括招聘目标总量、每日目标拆解和留存率目标。通常用于招聘目标制定、月度目标追踪、实际入培数据对比、渠道目标完成情况分析和基地目标达成复盘。

该表可与员工信息表中的实际入培数据配合使用：目标表提供计划目标，员工信息表提供实际入培人数。分析时可按 `年月份`、`基地`、`渠道` 与员工信息表中的入培月份、归类部门、招聘渠道进行对比。

## 3. 行数据说明

每一行代表某个月份下，一个 `基地 + 渠道 + 招聘订单类型` 组合的招聘目标配置。

例如：`2026-04 + 济南基地-济阳 + 渠道社招 + 内单` 表示 2026 年 4 月济南基地-济阳通过渠道社招产生的内单招聘目标。

## 5. 数据特征总结

- 该表是以目标配置为粒度的宽表，每行代表一个 `月份 + 基地 + 渠道 + 招聘订单类型` 的目标组合。
- 当前样例为 2026 年 4 月目标数据，`年月份` 在 Excel 中以日期序列 `46113` 存储。
- 当前样例共有 15 个基地、4 类招聘渠道、1 类招聘订单类型。
- `7天留存率目标`、`15天留存率目标`、`30天留存率目标` 当前为统一目标值，分别为 70%、65%、55%。
- `招聘目标` 是每日目标的汇总字段，公式为 `SUM(1日:31日)` 对应的行范围。
- `1日` 至 `31日` 是每日拆解目标，空值表示该日期未配置目标或按 0 参与月度汇总。
- 对比实际入培数据时，建议将该表转换为长表结构：保留 `年月份`、`基地`、`渠道`、`招聘订单类型`，将 `1日` 至 `31日` 展开为 `日期` 和 `每日招聘目标` 两列。

## 6. 与实际入培数据的匹配口径

- 时间匹配：目标表使用 `年月份` 和 `1日` 至 `31日` 生成目标日期；实际数据使用员工信息表中的 `入培时间`。
- 部门匹配：目标表使用 `基地`；实际数据使用员工信息表中按部门归类口径生成的部门字段。
- 渠道匹配：目标表使用 `渠道`；实际数据使用员工信息表中的 `招聘渠道`。
- 人数口径：实际入培人数建议按 `工号` 去重统计。
- 对比口径：可按 `月份 + 基地/部门 + 渠道` 对比月度目标与实际入培人数，也可展开到 `日期 + 基地/部门 + 渠道` 对比每日目标与实际入培人数。

### 6.1 一线员工所属基地聚合规则

一线员工列表中的 `所属基地` 必须先归一到目标表 `基地` 枚举后，再参与目标达成进度聚合。归一时优先使用员工表的 `部门` 和 `办公地点`，输出值以目标表中的基地名称为准。

如果数据来自组织表 `common_emp_org_real_day`，不能只使用 `org_level2` 或末级 `org_name` 作为基地。应先拼接 `org_path`、`org_name`、`org_level2` 至 `org_level7` 和 `work_location`，再按下表归一。原因是组织表常见结构为 `河北基地 > 10015升投 > 前台 > 培训期 > 前台20260623班`，末级 `org_name` 只保留班组名，直接匹配会丢失 `10015升投`、`前台`、`中台` 等关键层级。

### 6.2 一线员工对标目标规则

- 实际数据源：优先使用一线员工列表对应的组织表数据 `common_emp_org_real_day`。
- 一线岗位口径：`客服专员`、`客服班长`、`培训期学员`。
- 时间口径：使用 `train_start_date` / `入培时间` 归属统计月份；字段为日期对象或文本时，都必须先标准化为 `YYYY-MM-DD` 后再判断月份。
- 基地口径：使用 6.1 的聚合结果对标目标表 `基地`。
- 渠道口径：使用员工表 `hr_channel_type` / `招聘渠道` 对标目标表 `渠道`。
- 人数口径：按 `工号` 去重统计实际入培人数。
- 实际-only 渠道：同一目标基地内，如果某渠道没有目标行但当月有实际入培，例如 `回流` 或额外出现的 `渠道社招`，仍需补充为 `月度目标 = 0` 的渠道行，并计入该基地和整体的 `实际入培` 合计。
- 基地展示：目标和实际均为 0 的基地可隐藏；有目标或有实际任一非 0 的基地必须展示。

## 7. 目标达成计算口径

- 达成：实际入培人数。
- 目标：目标表中的 `招聘目标`，或按日期展开后的每日招聘目标。
- GAP：`达成 - 目标`。
- 达成率：`实际入培人数 / 招聘目标`。
- 当 `招聘目标 = 0` 或 `招聘目标` 为空时，达成率为空或不计算，避免除零。

## 8. 报表模板口径

- 统计截止日期：默认截止日期为当天。
- 月度目标：按每个自然月上传的目标表统计，月度目标为该自然月的完整招聘目标。
- 截止目标：从目标表已有的日度招聘目标中，累计 `1日` 至统计截止日期当天的目标和。
- 实际入培数：后台看板优先使用一线员工列表对应的组织表 `common_emp_org_real_day`，按 `train_start_date` / `入培时间` 统计；离线 Excel 报表可继续合并在职员工信息表和离职员工信息表。两类数据源均需按 `工号` 去重。
- 渠道顺序：报表中渠道固定按 `回流`、`内推`、`渠道社招`、`渠道校招`、`自主社招`、`合计` 排序。
- 合计行规则：同一个基地内的 `合计 = 回流 + 内推 + 渠道社招 + 渠道校招 + 自主社招`。
- 条件格式规则：`GAP >= 0` 显示绿色圆点，`GAP < 0` 显示红色圆点。
- 百分比展示规则：`达成率`、`渠道目标占比`、`渠道达成占比`、`占比GAP` 均按百分比格式展示，保留 2 位小数。
- 百分比换算规则：以上字段底层计算结果为小数，例如 `0.875` 表示 `87.50%`，`-0.05` 表示 `-5.00%`。
- 空值展示规则：当分母为 0 或为空时，展示为 `0.00%`。
