# 月度招聘达成进度

## 文件

- 输入：`人才开发目标拆解-X月-月日.xlsx`、`在职员工信息_*.xlsx`、`离职员工信息_*.xlsx`
- 输出：`月度招聘达成进度/月度招聘达成进度-X月.xlsx`
- 目标拆解归档：每月上传的 `人才开发目标拆解-X月-月日.xlsx` 需要封存到 `人才开发目标拆解归档/`，便于年度汇总。
- 结果留存：生成的月度招聘达成进度文件保留在 `月度招聘达成进度/`。
- 招聘负责人看板：基于已生成的月度结果表输出到 `招聘负责人看板/招聘负责人看板-X月.html`。
- 员工表文件名日期不参与计算；同目录有多份时必须明确使用哪一份。

## 核心口径

- 目标表 sheet: `整体目标`
- 月度目标：`1日` 到当月最后一天之和
- 截止目标：`1日` 到统计截止日之和；当前月默认当天，历史月默认月末
- 实际入培：合并在职 + 离职，按 `入培时间` 筛选，按 `工号` 去重
- 渠道顺序：`回流`、`内推`、`渠道社招`、`渠道校招`、`自主社招`、`合计`
- 基地别名：`15升投` -> `10015升投`

## 部门归类

- `京津冀大区`：取二级部门，去掉 `京津冀大区` 前缀，例如 `联通河北`
- `长春基地`：取二级部门，去掉 `长春基地` 前缀，例如 `长春热线项目`
- `江苏基地`：取三级部门前两个字，例如 `江苏基地-淮安`、`江苏基地-南京`
- `湖南基地`：按 `办公地点` 或部门路径归为 `湖南基地-空港`、`湖南基地-荷花`
- `济南基地`：按 `办公地点` 或部门路径归为 `济南基地-济阳`、`济南基地-夏都`
- 其余部门取 `OPS` 后第一级

## 计算公式

- `GAP = 实际入培数 - 月度目标`
- `达成率 = 实际入培数 / 月度目标`
- `渠道目标占比 = 渠道月度目标 / 基地合计月度目标`
- `渠道达成占比 = 渠道实际入培数 / 基地合计实际入培数`
- `占比GAP = 渠道达成占比 - 渠道目标占比`

当分母为 `0` 或为空时，百分比字段展示为 `0.00%`。

## 人效核算

- 工作表：`人效核算`
- 招聘规模：部门含 `人才开发部` 且职位为 `招聘专员`，统计月内有在岗交集
- 自主社招归属：`招聘渠道 = 自主社招`，`渠道名称` 取纯姓名；`姓名+工号` 取 `+` 前姓名
- 阶段：统计月最后一天入职满 6 个月为 `正式期`，否则 `试用期`
- 标准目标：正式期参培 `20`、7天 `12`；试用期参培 `12`、7天 `8`
- 入离职按在岗天数折比，目标四舍五入为整数
- 7天达成：`7天发生日期 = 入培时间 + 6天`；参培天数含首尾，满 7 天才计入
- 人效统计：`试用期`、`正式期`、`整体`；人效 = 实际达成 / 招聘规模，保留 1 位小数，不显示百分比
- `初级招聘主管` 达成只加入 `整体` 达成，招聘规模不计入主管人数

## 展示格式

- 第 1 行直接作为标题行，不生成前置说明行。
- `达成进度` 顶部加 `整体` 分组；每个基地按固定渠道顺序展示 6 行；同一基地纵向合并。
- 全表网格线、居中；标题行加粗、蓝底、黑字；`基地` 列加粗。
- 百分比字段 `xx.xx%`；`达成率 < 100%` 标红，`< 70%` 标红加粗。

## 招聘负责人看板

- 顶部预览：整体目标、截止目标、实际入培数、GAP、达成率、未达成基地数、自主社招整体人效、自主社招占比。
- 日期筛选：顶部提供统计口径、年份、月份筛选控件；月度视图展示截止目标。
- 全局渠道占比：顶部卡片下方展示各渠道目标、截止目标、实际、目标占比、达成占比、占比 GAP。
- 各基地渠道达成明细：在达成总览下先展示 `整体达成`，再展示每个基地各渠道的月度目标、截止目标、实际入培、GAP、达成率、渠道占比；同一基地纵向合并展示，达成率不足 `100%` 时标红。
- 基地风险：展示所有有目标或有达成数据的基地；目标和达成都为 0 的基地隐藏；负 `GAP` 基地标红。
- 未达成基地：只分析未达成基地，按渠道 `GAP` 缺口输出 TOP3；点击 `实际入培` 数字后在二级弹窗页展示 `所属基地`、`工号`、`姓名`、`手机号`、`招聘渠道`、`渠道名称`、`入培日期`、`在职状态`、`离职日期`、`在职天数`。
- 自主社招：重点展示阶段汇总人效和招聘人员明细；点击人效汇总中的 `参培达成` 或 `7天达成` 数字后在二级弹窗页展示人员达成数据。

## 执行命令

在项目根目录运行：

一键生成指定月份的月度结果表和招聘负责人看板（推荐复用入口）：

```bash
python .cursor/skills/recruitment-progress-report/scripts/generate_month_dashboard.py \
  --year 2026 \
  --month 6
```

固定读取规则：

- 目标表：`人才开发目标拆解归档/人才开发目标拆解-X月-*.xlsx`
- 在职员工表：根目录最新的 `在职员工信息_*.xlsx`
- 离职员工表：根目录最新的 `离职员工信息_*.xlsx`
- 月度结果表输出：`月度招聘达成进度/月度招聘达成进度-X月.xlsx`
- 招聘负责人看板输出：`招聘负责人看板/招聘负责人看板-X月.html`
- 生成指定月份结果表后，会重建 `月度招聘达成进度/` 目录下所有已有月份的看板；顶部月份筛选可在已有月份看板之间切换。
- 固定入口网页应用：`招聘负责人看板/index.html`，页面内支持 `单月 / 全年` 切换。
- 网页应用数据：`招聘负责人看板/data/months.json`、`招聘负责人看板/data/YYYY-MM.json`

本地打开固定入口时，建议在项目根目录启动本地服务：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000/招聘负责人看板/
```

需要指定截止日时添加 `--cutoff-day 23`。

单独生成月度结果表：

```bash
python .cursor/skills/recruitment-progress-report/scripts/generate_report.py \
  --target "人才开发目标拆解-5月-0622.xlsx" \
  --active "在职员工信息_20260620.xlsx" \
  --leave "离职员工信息_20260620.xlsx" \
  --year 2026 \
  --month 5
```

默认输出到 `月度招聘达成进度/月度招聘达成进度-X月.xlsx`。需要指定截止日时添加 `--cutoff-day 23`。

单独生成招聘负责人看板：

```bash
python .cursor/skills/recruitment-progress-report/scripts/generate_dashboard.py \
  --report "月度招聘达成进度/月度招聘达成进度-6月.xlsx" \
  --active "在职员工信息_20260620.xlsx" \
  --leave "离职员工信息_20260620.xlsx" \
  --year 2026 \
  --month 6
```

默认输出到 `招聘负责人看板/招聘负责人看板-X月.html`。

## 验证

- 检查脚本输出的整体 `target`、`actual`、`gap`、`achievement_rate`。
- 输出文件应包含 `达成进度`、`人效核算`、`汇总`、`未匹配实际数据`。
- 如果 `未匹配实际数据` 非 0，向用户说明哪些实际基地没有目标。
- 招聘负责人看板应包含顶部筛选、顶部预览、全局渠道占比、基地风险、未达成基地、自主社招人效汇总和人员明细。
